# @fuggetlenfe/brand-styles

Visual identity packs that implement the token contract with concrete brand and theme values. These CSS files are the final layer that gives components their appearance.

## Architecture role

```text
@fuggetlenfe/tokens/contract.css  defines --ff-button-bg-default (stable API)
                                           |
@fuggetlenfe/brand-styles         overrides that variable per brand+theme
  brand-1-light.css               [data-brand="brand-1"][data-theme="light"]
  brand-1-dark.css                [data-brand="brand-1"][data-theme="dark"]
  brand-2-light.css               ...
  ...
                                           |
                                           v
Consumer app shell                sets data-brand and data-theme attributes
  <main data-brand="brand-1"     which activates the matching CSS overrides
        data-theme="light">
```

## How brand activation works

1. The consumer app imports `contract.css` (base values) and one or more brand pack CSS files.
2. Brand packs use attribute selectors like `[data-brand='brand-1'][data-theme='light']` to override token values.
3. The consumer app shell sets `data-brand` and `data-theme` on a parent element.
4. CSS cascade activates the matching overrides, and every component inside that shell receives the brand identity automatically.

No JavaScript is needed for brand switching. It is pure CSS cascade.

## Consumer-owned packs

These are not generated from Figma. They demonstrate that any consumer can create their own brand pack that implements the same token contract.

| File | Purpose |
|---|---|
| `custom-brand-light.css` | Custom brand, light theme |
| `custom-brand-dark.css` | Custom brand, dark theme |
| `custom-brand.css` | Custom brand base values |
| `demo.css` | Demo/documentation styles |

## Import pattern

```css
/* Angular: in styles.css */
@import '@fuggetlenfe/tokens/contract.css';
@import '@fuggetlenfe/brand-styles/brand-1-light.css';
@import '@fuggetlenfe/brand-styles/brand-1-dark.css';
```

```tsx
/* React: in main.tsx */
import '@fuggetlenfe/tokens/contract.css';
import '@fuggetlenfe/brand-styles/brand-2-light.css';
import '@fuggetlenfe/brand-styles/brand-2-dark.css';
```

## What must not go here

- Component logic or structure
- JavaScript brand switching
- Consumer-specific layout rules
